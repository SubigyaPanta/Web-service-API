User

A user profile in the application.
The default user for the application is root and it has the user_id=1.

Example

http://ws-api.cloudaccess.net/ws/www/users/1 (the root user)

Fields

The User object has the following fields:

Name Description Returns
id The user ID integer
name The user's full name string
username The user's JWS username string
password The user's password
Note: Since we don't make any encoding to the user password we do strongly recommend to use a safe encoding method such as SHA1 before saving the user's password 		string
email The user's email string
user_id The user ID that created this user integer
created_at The created date A struncture with date, timezone_type and timezone
last_visit The last visit date of the user
 date
user_type The user's type string
activation An activation code that you can set in order to activate the user by email string
hits Hits count for this user integer

GET

GET requests can be made to /users in order to get all users or to /users/:user_id in order to get information about a specific user.

Field Description Default
fields A string with fields separated by comma with the fields that are returned. We recommend to request only the necessary fields in order to save time. All
order A string with fields separated by comma with the sort order of the returned objects. ID
since A string in date format (yyyy-MM-dd) representing the minimum created date 01-01-1970
before A string in date format (yyyy-MM-dd) representing the maximum created date now
offset An integer representing the first result 0
limit An integer representing the number of results
Note: The maximum results that can be returned are 100. 		20
