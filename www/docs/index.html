Concepts

JWS API (Joomla Web Service API) presents a simple way of creating a RESTful API using the Joomla! Framework and the new Unified Content Model. The current version of the API is 1.0 and developers can create directory-like applications with it.

Every object in the API has a unique ID and it is considered beeing content. The properties of an object can be accessed making a request at http://ws-api.cloudaccess.net/ws/www/RESOURCE/ID.

The default output format is JSON and for the moment is the only supported format, but in the future XML will also be available. The output format can be chosen by adding .json or .xml at the end of the resource path.

The HTTP methods supported are: GET, POST, PUT, DELETE. If your client doesn't support advanced methods such as PUT or DELETE you can make PUT or DELETE requests by making a POST request and adding _method=HTTP_METHOD in the query.

The version of the API should be specified at the highest level scope (http://ws-api.cloudaccess.net/ws/www/v#) in the URL having a prefix v and an integer representing the version. For the moment, the only available and default version is v1.

By default the response code is 200 for successful result and 400 for failure. Passing the parameter suppress_response_codes=true all responses will have the successful code 200. By default, failure returns a json structure with information about the error occured.
